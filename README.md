# Cohesive

Local-first desktop coding agent for exploring and editing real projects.

Cohesive is a Tauri 2 app with an Explore/Build agent loop, permissioned tools, and inspectable traces. Data stays on your machine; AI calls go through your own DeepSeek API key.

> Writing and Mind domains exist in the codebase but are currently hidden from the product UI while Code is the showcase surface.

## Why this repo

Built as a portfolio-grade agent product: not a chat demo, but a desktop workspace with modes, tool permissions, persistence, and an eval harness around the agent loop.

## Quick start

```bash
npm install
npm run tauri dev
```

Requirements: Node 22+, Rust stable, macOS (primary target).

On first launch, add a DeepSeek API key in onboarding or Settings.

## Product surface

- **Explore** — read-only agent: search/read the codebase and explain
- **Build** — mutating agent: edit files and run commands with explicit permissions
- **Trace panel** — timeline of tool calls for the current run
- **Light / dark theme** — default light; toggle in the top bar or Settings

## Scripts

```bash
npm run tauri dev          # desktop app
npm run typecheck          # app + eval TS
npm run eval:unit          # offline agent loop tests
npm run eval:live          # live LLM evals (needs API key)
npm run website:dev        # landing page
npm run tauri:build:arm64  # macOS Apple Silicon
npm run tauri:build:x64    # macOS Intel
```

## Landing page

Static site in `website/`. Deploy the `website/` folder to Vercel (or any static host).

```bash
npm run website:dev
npm run website:build
```

Update download / GitHub URLs in `website/src/config.ts` before publishing.

## App icon

Source mark is generated by a zero-dependency Node script (shared geometry → PNG + SVG), then exported through Tauri:

```bash
npm run icon:generate   # writes assets/brand/icon-1024.png + icon.svg
npm run icon:svg        # SVG only
npm run icon:export     # generate + tauri icon → src-tauri/icons/*
```

In-app `BrandMark` inlines the generated geometry and colors it with
`--brand-mark-from` / `--brand-mark-to` / `--brand-mark-fg` so it follows
light/dark theme. `assets/brand/icon.svg` stays a static export for landing
pages and marketing (CSS variables do not work inside `<img>`).

## Releases

Push a version tag to trigger macOS arm64 + x64 builds:

```bash
git tag v0.1.0
git push origin v0.1.0
```

GitHub Actions (`.github/workflows/release.yml`) creates a draft Release with Tauri artifacts.

Keep versions in sync across:

- `package.json`
- `src-tauri/tauri.conf.json`
- `src-tauri/Cargo.toml`
- `website/src/config.ts`
